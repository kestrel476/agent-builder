# Экспертиза банковской гарантии — Спецификация

- **Идентификатор:** `ekspertiza-bankovskoy-garantii`
- **Архетип:** checklist
- **Версия:** 1
- **Этап:** tested
- **Возможности:** llm, doc_extraction, ocr, rules, json_output, multi_step, logging

## Цель
Проведение правовой экспертизы банковской гарантии путем сравнения условий текста гарантии с контрольным списком (чек-листом).

## Входные данные
| имя | тип | обязателен | форматы |
|---|---|---|---|
| document | document | да | pdf, docx, txt, eml, png |

## Контракт выхода
Тип: **report**

| поле | тип | обязательно | описание |
|---|---|---|---|
| `results` | array | да | Результат по каждой проверке чек-листа. |
| `flagged` | array | нет | Коды сработавших проверок (verdict=yes). |
| `overall_position` | string | да | Итоговая позиция по документу. |

## Критерии успеха
- Все обязательные поля присутствуют и непусты: results, overall_position.
- Значения полей прослеживаются до исходного документа (без выдумок).

## Политика ошибок / частичного результата
ERROR: вход нечитаем или ни одно обязательное поле невозможно определить. PARTIAL: часть обязательных полей отсутствует (выводятся как null). LOW_CONFIDENCE: результат ниже порога уверенности.

Выводи каждое поле, которое можно определить; неопределимые поля задавай как null; никогда не отбрасывай оболочку результата. Отмечай частичные результаты в `status`.
