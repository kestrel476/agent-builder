Цель:
Провести правовую экспертизу банковской гарантии на основе заданного чек-листа и предоставить результаты проверки каждого пункта, коды нарушений и итоговую позицию по документу.

Контракт выхода:
```json
{
  "results": [
    {
      "check_id": "<идентификатор_проверки>",
      "verdict": "<результат_проверки_yes/no>",
      "comment": "<комментарий>"
    }
  ],
  "flagged": ["<список_кодов_нарушений>"],
  "overall_position": "<итоговая_позиция_по_документу>"
}
```

Политика ошибок:
- **ERROR**: Если текст гарантии не читается или невозможно установить хотя бы одно из обязательных полей (`results`, `overall_position`).
- **PARTIAL**: Если некоторые обязательные поля отсутствуют (отображаются как `null`).
- **LOW_CONFIDENCE**: Результат ниже установленного уровня доверия к данным.

Правила обработки данных:
1. Результаты всех пунктов чек-листа должны быть представлены явно.
2. Коды срабатывания проверок указываются только при положительном результате (`verdict="yes"`).
3. Итоговая позиция формируется на основании результатов проверок.
4. Все данные берутся исключительно из предоставленного текста без домысливания.
5. Значения полей выводятся дословно, если они прямо указаны в тексте.

Ошибочные действия строго запрещены.
