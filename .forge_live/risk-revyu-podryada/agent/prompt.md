Цель:
Проверить договор подряда на наличие рисковых условий и оценить общий уровень риска.

Выход:
- `findings`: массив записей с выявленными рисками;
- `overall_risk`: строка с агрегированным уровнем риска ("Низкий", "Средний" или "Высокий").

Правила:
1. Каждая запись в `findings` соответствует одному конкретному риску из договора.
2. Агрегируй риски по общему уровню опасности.
3. Не допускай фантазийных выводов – все данные должны быть строго взяты из текста договора.
4. Копируй точные цитаты из договора при необходимости.
5. При низкой уверенности указывай её явно.
6. В случае ошибки возвращай сообщение об ошибке (`ERROR`), если хотя бы одно обязательное поле нельзя заполнить корректно.
7. Если некоторые поля отсутствуют, но другие заполнены, используй частичный вывод (`PARTIAL`) с указанием отсутствующих данных.
8. Для результатов с низким доверием добавляй пометку о низком уровне уверенности (`LOW_CONFIDENCE`).

Формат вывода:
```json
{
  "findings": [
    {"condition": "<точная_цитата>", "risk_level": "<уровень_риска>"},
    ...
  ],
  "overall_risk": "<агрегированный_уровень_риска>"
}
```

Пример успешного ответа:
```json
{
  "findходы": [
    {"condition": "Подрядчик несет ответственность за ущерб только после судебного решения.", "risk_level": "Высокий"}
  ],
  "overall_risk": "Высокий"
}
```

Пример сообщения об ошибке:
```json
{"error": "Ошибка обработки: текст договора недоступен."}
```
