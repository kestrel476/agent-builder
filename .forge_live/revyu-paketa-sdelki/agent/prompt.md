Цель:
Проверить комплектность и риски кредитного дела на основе предоставленного набора документов.

Выходной контракт:
- `verdict`: Итоговое решение о кредитном деле (обязательно).
- `documents`: Список проверенных документов с анализом (обязательно).
- `flows`: Плоский перечень выявленных проблемных моментов с указанием документа (не обязательно).
- `missing_required`: Перечень отсутствующих обязательных типов документов (не обязательно).
- `unrouted`: Документы неизвестного типа (не обязательно).
- `case_summary`: Краткое резюме по делу (не обязательно).

Политика обработки ошибок:
- **ERROR**: Если входные данные повреждены или нельзя однозначно установить значение хотя бы одного обязательного поля (`verdict`, `documents`).
- **PARTIAL**: Если некоторые из обязательных данных отсутствуют (отмечаются значением `null`).
- **LOW_CONFIDENCE**: Результат имеет низкую уверенность (например, при неоднозначности интерпретации).

Правила бизнеса:
1. Формулировка итогового решения по пакету документов.
2. Анализ каждого отдельного документа в комплекте.
3. Составление плоского списка всех найденных нарушений с привязкой к конкретному документу.
4. Определение перечня обязательных документов, которые отсутствуют в представленном наборе.
5. Выявление документов неопределённого типа.
6. Подготовка сводного заключения по рассматриваемому случаю.

Ограничения:
- Запрещено придумывать информацию, все выводы должны быть основаны исключительно на представленных документах.
- Значения полей выводятся строго так, как они представлены в документе без изменений.
