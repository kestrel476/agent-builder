# Извлечение реквизитов счёта — Спецификация

- **Идентификатор:** `izvlechenie-rekvizitov-scheta`
- **Архетип:** json_extraction
- **Версия:** 1
- **Этап:** tested
- **Возможности:** llm, doc_extraction, ocr, json_output, schema_validation, confidence, logging

## Цель
Извлечение ключевых реквизитов из счетов на оплату и представление их в формате JSON.

## Входные данные
| имя | тип | обязателен | форматы |
|---|---|---|---|
| document | document | да | pdf, docx, txt, eml, png |

## Контракт выхода
Тип: **json**

| поле | тип | обязательно | описание |
|---|---|---|---|
| `supplier_name` | string | да | Наименование поставщика |
| `supplier_inn` | string | да | ИНН поставщика |
| `invoice_number` | string | да | Номер счёта |
| `invoice_date` | string | да | Дата счёта |
| `total_amount` | number | да | Итоговая сумма |
| `vat_amount` | number | нет | Сумма НДС или null, если отсутствует |

## Критерии успеха
- Все обязательные поля присутствуют и непусты: supplier_name, supplier_inn, invoice_number, invoice_date, total_amount.
- Значения полей прослеживаются до исходного документа (без выдумок).
- Результат сопровождается калиброванной оценкой уверенности.

## Политика ошибок / частичного результата
ERROR: вход нечитаем или ни одно обязательное поле невозможно определить. PARTIAL: часть обязательных полей отсутствует (выводятся как null). LOW_CONFIDENCE: результат ниже порога уверенности.

Выводи каждое поле, которое можно определить; неопределимые поля задавай как null; никогда не отбрасывай оболочку результата. Отмечай частичные результаты в `status`.
