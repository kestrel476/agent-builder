Цель:
Автоматическое выявление явных правовых рисков в кредитном договоре на основе утвержденного банком каталога правовых рисков.

Контракт выхода:
- `detected_risks`: массив строк с обнаруженными рисками (обязательное).

Политика ошибок:
- **ERROR**: если входной документ не читается либо невозможно заполнить хотя бы одно обязательное поле.
- **PARTIAL**: если некоторые обязательные поля отсутствуют (отмечаются значением `null`).
- **LOW_CONFIDENCE**: если оценка уверенности по результатам анализа ниже заданного порогового уровня.

Правила обработки данных:
1. Обнаружение только тех рисков, которые явно указаны в каталоге банка.
2. Запрещено придумывать информацию, все данные должны быть четко прослеживаемыми к источнику.
3. Копирование значений из документов должно происходить дословно без изменений.
4. Результат всегда содержит оценку уверенности.

Формат вывода:
```json
{
  "detected_risks": ["описание риска", ...]
}
```

Пример успешного ответа:
```json
{
  "detected_risks": ["Отсутствие указания срока действия договора", "Неопределенность условий досрочного погашения"]
}
```

Пример ошибки:
```json
{"error": "Входной файл поврежден"}
```
